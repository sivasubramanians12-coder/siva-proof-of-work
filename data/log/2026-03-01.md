# Shipped proof-of-work site — Next.js static, Vercel, full brand system

Built and deployed this site today. Notes on the stack and decisions.

**Stack:**
- Next.js 14 App Router with TypeScript strict mode
- Static generation (`generateStaticParams`) — no runtime server, pure static HTML
- Tailwind + CSS variables for the brand system
- Data in TypeScript files (`data/cases.ts`) — no CMS, no database
- Deploy: Vercel, auto-deploys on GitHub push to main

**Brand system implementation:**
CSS variables set in `:root` and applied throughout. Key ones:
```css
--ink: #1a1a1a;
--paper: #f5f2eb;        /* warm cream, not white */
--signal-amber: #c4841d;
--evidence-green: #3a6b4a;
--draft-blue: #2d5f8a;
--type-display: 'Instrument Serif';
--type-mono: 'IBM Plex Mono';
--type-body: 'DM Sans';
```
Grid paper texture via `background-image` with `rgba(26,26,26,0.03)` — subtle but effective.

**What Claude Code built in one session:**
- Full Next.js scaffold with App Router
- Hero, Nav, Footer, CaseCard components
- `/` home with case feed
- `/cases/[slug]` dynamic case pages
- `/log` daily build log
- All 5 cases with full content

**Decisions:**
- No MDX for now — TypeScript data files are simpler to maintain and type-safe
- Static generation over SSR — no dynamic data yet, static is faster and free on Vercel
- CSS variables over Tailwind config extension — easier to reference in inline styles for complex components

**What's next:**
- Pull case data from `siva-builds` GitHub repo via API (currently hardcoded in `data/cases.ts`)
- Add GitHub commit activity to /log automatically
- Custom domain once decided
