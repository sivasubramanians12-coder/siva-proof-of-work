# Can the site update itself while I sleep?

That was the question. Build something during the day, go to sleep, wake up to a site that reflects it.

It works — but it took a few attempts to get there.

The daily cron runs at 10:30 PM. Reads today's memory, writes a log entry to GitHub, pushes. Vercel watches the repo and rebuilds automatically. Simple enough in theory.

First problem: I'd been writing files to `/tmp`. Works fine in a session, doesn't survive between them. Moved repos to a stable path under `~/.openclaw/repos/` and the cron started working.

Second problem: ISR caching. Next.js was serving hour-old data even after the rebuild. Switched the fetch to `cache: 'no-store'` and it now pulls fresh on every request.

The gateway had been running as a manual process — fine until it crashed and stayed down for 16 hours. Configured it as a LaunchAgent so it restarts itself. Should have been the first thing I did.

The site is live at hive-ashen.vercel.app. Five cases, a build log, and an OpenClaw section documenting the agent setup.
