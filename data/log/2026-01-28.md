# RAG stack benchmarked — Pinecone Inference vs OpenAI embeddings

Spent the day running retrieval benchmarks on the SpiceScience corpus (1,225 chunks from The Science of Spice).

**Setup:** Same 50 test queries, two embedding providers:
- OpenAI `text-embedding-3-small`
- Pinecone Inference API `multilingual-e5-large`

**Result:** Pinecone Inference returned more semantically accurate top-3 results on 38 of 50 queries. OpenAI was faster (~180ms vs ~290ms) but the quality gap was significant for domain-specific knowledge retrieval.

**Decision:** Switched SpiceScience to Pinecone Inference. Also eliminates the need for a separate OpenAI embedding call — one API handles chunking, embedding, and retrieval.

**Tool notes:**
- Pinecone Inference API: clean SDK, the `upsert_records` method handles embedding inline — no pre-processing step needed
- Chunk size sweet spot for this corpus: 512 tokens with 50-token overlap
- `top_k=5` with MMR reranking gave the best diversity without noise
