# Switching embedding providers mid-project

I'd already built SpiceScience on OpenAI embeddings when someone mentioned Pinecone's own inference API. Figured I'd benchmark before shipping.

Same 50 test queries, both providers. Pinecone's `multilingual-e5-large` returned better top-3 results on 38 of them. OpenAI was faster but the quality gap was real — especially on domain-specific queries about spice compounds.

The switch took a few hours. The bigger lesson: chunk size matters more than which provider you pick. I'd been chunking at arbitrary character limits. Moving to 512 tokens with semantic boundaries made both providers better. The embedding model is almost secondary.

One less API to manage too. Pinecone handles embedding and retrieval in one call.
