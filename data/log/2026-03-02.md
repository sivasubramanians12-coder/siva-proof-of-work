# WalletBot ‚Äî family finance via Telegram

Built WalletBot: a two-person family finance system where expense logging happens in natural language over Telegram and Claude classifies everything automatically.

**Architecture ‚Äî pnpm monorepo:**
```
apps/bot     ‚Äî Telegraf.js serverless function on Vercel
apps/web     ‚Äî Next.js 15 + shadcn/ui + Recharts dashboard
packages/db  ‚Äî Turso (libSQL) client + all queries
packages/shared ‚Äî Types, constants (BUCKETS, formatINR), utilities
```

The monorepo pattern keeps types in sync across bot and dashboard without duplication. `@walletbot/db` and `@walletbot/shared` are internal packages ‚Äî any change to a query or type propagates everywhere. Switched from Supabase ‚Üí Turso mid-build; `fs.readFileSync` doesn't work on Vercel serverless, so a cloud DB was mandatory from the start.

**The Claude classification layer:**
`apps/bot/src/classify.ts` handles the core intelligence ‚Äî takes a raw message like "Zomato 850" or "spent 1200 on petrol" and returns `{ bucket: 'Dining', amount: 850, description: 'Zomato' }`. Rule-based classification breaks on Indian merchant names. Claude doesn't. 14 buckets total including Credit Card.

**Savings model ‚Äî explicit, not derived:**
Key design decision: savings = explicit `/save` commands, not (income - expenses). Misc gap = income - expenses - savings, shown as an amber banner. Investments have two concepts: corpus (amount saved) vs current_value (market). `/save invest` updates corpus; `/invest update` only updates market value. This distinction matters when tracking MF/stocks/PF.

**WalletBot agent removed from openclaw.json:**
The walletbot agent was running as a separate OpenClaw agent, causing config errors and routing confusion. Removed all three blocks (agent definition, binding, telegram account), fixed a trailing comma bug, and restarted gateway cleanly. BarleyBot now handles the WalletBot group directly via WALLETBOT.md routing ‚Äî simpler and it works.

# WalletBot v2 ‚Äî "Lifestyle Finance Magazine" redesign

Pushed v2 redesign to `feature/v2-redesign` branch. Design pivot from neutral sand to editorial:

- **Fonts:** Fraunces (display) + Plus Jakarta Sans (body) ‚Äî replaces DM Serif + Outfit
- **Colors:** Deep green `#1B6B3A` dominant, terracotta `#C4654A` for Gayatri
- **Hero:** Full-width gradient banner with frosted stat pills (balance, income, savings)
- **Story cards:** 3 gradient cards (dark/terracotta/teal) replacing pulse cards
- **Goal cards:** Cycling top bar colors, Fraunces 24px percentage, contribution tracking
- **Investment cards:** Colored top bars per type (MF=green, stocks=teal, PF=mustard)
- **Charts:** Siva=#22A352, Gayatri=#C4654A, grid=#F3EDE3

**One build bug caught:** `onMouseEnter`/`onMouseLeave` handlers in a Next.js server component (`async` function in `app/` dir) don't throw at build time ‚Äî they throw at runtime. Fix: extracted story cards into a `'use client'` component using CSS `:hover` instead. This is a hard rule to burn in: event handlers must live in client components.

**Coder subagent lesson:** Ran 3 parallel Coder sessions on this build ‚Äî all failed (overload errors, git conflicts). Applied all changes directly via read/write/edit tools instead. For multi-file edits across a known codebase, doing it directly is faster than spawning Coder when there's any instability.
