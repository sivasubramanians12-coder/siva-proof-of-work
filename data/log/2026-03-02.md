# Building WalletBot — family finance over Telegram

The problem with most finance apps is adoption. You start strong and abandon in week three.

The insight for WalletBot: don't build a new habit. Use the one that already exists. Both of us are in Telegram constantly. If expense logging is just sending a message — "Zomato 850", "groceries 1200", "petrol 500" — the friction is low enough to stick.

Claude classifies every message into one of 13 buckets automatically. No dropdowns, no categories to remember. It handles merchant names, typos, shorthand. Rule-based systems break on Indian merchant names. Claude doesn't.

The architecture is a pnpm monorepo: Telegraf.js bot deployed as a serverless function, Next.js dashboard for trends, shared database package with all the Supabase queries. Keeping types in sync between bot and dashboard without the monorepo would have been a maintenance headache.

Two users, one shared ledger. Goals with target amounts. `/summary` for a quick month view. `/undo` because mistakes happen.

Still early — but the basic loop works.
