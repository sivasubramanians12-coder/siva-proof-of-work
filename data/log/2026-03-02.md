# WalletBot — family finance via Telegram

Built WalletBot: a two-person family finance system where expense logging happens in natural language over Telegram and Claude classifies everything automatically.

**Architecture — pnpm monorepo:**
```
apps/bot     — Telegraf.js serverless function on Vercel
apps/web     — Next.js 15 + shadcn/ui + Recharts dashboard
packages/db  — Supabase client + all queries
packages/shared — Types, constants (BUCKETS, formatINR), utilities
```

The monorepo pattern keeps types in sync across bot and dashboard without duplication. `@walletbot/db` and `@walletbot/shared` are internal packages — any change to a query or type propagates everywhere.

**The Claude classification layer:**
`apps/bot/src/classify.ts` handles the core intelligence — takes a raw message like "Zomato 850" or "spent 1200 on petrol" and returns `{ bucket: 'Dining', amount: 850, description: 'Zomato' }`. Rule-based classification breaks on Indian merchant names. Claude doesn't.

**Natural language parsing:**
Messages that aren't commands get run through a simple regex first (amount + keyword pattern). If that matches, classify. If not, ignore. Keeps noise low — group messages, reactions, etc. don't trigger false positives.

**Supabase schema:**
- `users` — telegram_id, name
- `transactions` — user_id, amount, bucket, description, month, year
- `salaries` — user_id, amount, month, year  
- `goals` — name, target_amount
- `goal_allocations` — goal_id, user_id, amount, month, year

RLS policies scope transactions per user. Both users see aggregated month view (no RLS filter on reads) — intentional, it's a shared ledger.

**Dashboard pages:**
- `/` — monthly P&L, bucket breakdown
- `/transactions` — full ledger with filters
- `/goals` — goal cards with progress bars (Recharts)
- `/trends` — month-over-month by bucket
