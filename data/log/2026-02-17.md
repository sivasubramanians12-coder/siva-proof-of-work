# ElevenLabs Conversational AI — integration notes

Integrating ElevenLabs into HD Hire for the voice screening step. One agent handles all screening calls — prompt is overridden per call to inject the specific job requirements.

**Architecture:**
- Single ElevenLabs agent (one `ELEVENLABS_AGENT_ID`)
- Per-call `conversation_config_override` injects: role title, required skills, key questions
- Cal.com webhook triggers the call when a candidate books a slot
- Transcript webhook fires to our API when the call ends — stored in Supabase, scored by Claude

**What the docs don't say clearly:**
- `conversation_config_override` replaces the entire system prompt — you need to include your base instructions plus the role-specific additions, not just the additions
- The transcript webhook can fire before the call audio is fully processed — add a 30s delay before calling `get_conversation()` for the full transcript
- Voice latency is noticeable on Indian mobile networks (~400ms). Tested with shorter agent utterances — helps significantly

**Tool notes:**
- `elevenlabs` Python SDK v1.x — the `ConversationalAI` class handles most of this cleanly
- `ngrok` for local webhook testing — `elevenlabs.io/app/conversational-ai` lets you set a per-agent webhook URL
- Call duration average: 4–7 minutes for a screening call. Cost: ~$0.08–0.12 per call at current pricing
