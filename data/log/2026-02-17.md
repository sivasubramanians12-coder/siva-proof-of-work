# ElevenLabs voice screening — what the SDK doesn't tell you

The idea: candidates book a slot via Cal.com, ElevenLabs calls them for a 5-minute screen, transcript comes back to us scored by Claude. No human needed for the first conversation.

Getting it working took longer than expected. Two things the docs don't make clear:

`conversation_config_override` replaces the entire system prompt — not just the parts you add. I was injecting role-specific details assuming the base instructions would stay. They didn't. Had to restructure to send base + override together.

The transcript webhook fires before the audio is fully processed. First few calls came back empty. Adding a 30-second delay before fetching the full transcript fixed it.

Voice latency on Indian mobile networks is noticeable. Shorter agent utterances helped. Candidates adapted faster than I expected — a 4-minute AI phone screen is apparently not as weird as it sounds in 2026.
