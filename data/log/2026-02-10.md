# Supabase RLS — the thing the docs don't warn you about

Built Geneva's role-based access. Field reps should only see their own deals. Managers see their region. Simple enough.

Except every rep was seeing the full pipeline.

Turned out I was initialising the Supabase client once at startup with the service key. Service key bypasses RLS entirely — it's superuser access. The docs mention this but don't make it obvious enough.

The fix: initialise a new client per request, using the user's JWT as the auth header. That client respects row-level security properly. `auth.uid() = rep_id` then actually does what you expect.

It's one of those things that's obvious in retrospect and invisible until you've been bitten by it. Logging it here so future-me doesn't spend another day on this.
