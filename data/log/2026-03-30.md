title: Command Center v4 — From Broken Orchestration to a Functioning Build Intelligence Layer
tags: [ops, product, infra]
---

The core problem this week was simple but painful: the Command Center couldn't reliably orchestrate Claude Code builds. The engine was passing credentials wrong, context was going stale on restart, and prompts were hitting shell argument length limits on larger jobs. None of these are abstract engineering problems — they were blocking every automated build run and making the orchestration platform unreliable for real work.

Today's fixes addressed all three root causes. The engine now strips `ANTHROPIC_API_KEY` before spawning Claude Code (it was silently overriding stored auth), rebuilds `contextPreamble` fresh on every restart instead of carrying stale context, and pipes build prompts via stdin/file rather than shell args — eliminating truncation on larger PRDs. These are the kinds of fixes that don't show up in demos but are the difference between a platform that works and one that intermittently fails in production.

On top of the stability work, Command Center v4 shipped a significant UX and intelligence layer: a files page overhaul with categories, markdown preview, and Obsidian-linked knowledge navigation; a cost intelligence module showing per-agent spend and cumulative savings estimates; a projects + OKR framework that parses PROJECT.md files and renders OKR progress; and a refreshed dashboard with system vitals and an activity feed. The breadcrumb navigation, smart search, and recent sidebar round out the files experience.

The sprint merge (6 sprints into v4) also included unified state endpoints, SSE streaming for live run updates, and guardrails on the context chain. A docs site scaffolded in Next.js 16 was also pushed today as the public face for the Command Center. The platform is now stable enough to run real multi-agent orchestration jobs without manual intervention.
