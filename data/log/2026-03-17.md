SiteCapture went from scaffold to near-complete in a single day. 15 commits. Sprints v2 through v8 shipped.

**What shipped:**

CX core: project creation, JID lookup, financial data pull from Ops OS orders table, project status board, team-wide access (any CX member can act on any project without bottlenecks).

Scheduler: split-panel view for CX and coordinator. Assign CM visits and installation visits independently. Gap and exception handling. The same person pool covers both CM and installation — the system treats them as separate tasks, not separate roles.

Check Measure Capture (CM): Window-by-window capture form for the installer. Photos per window, product confirmation, DC observations, final measurements. Auto-save to IndexedDB — the form never blocks on the server. Photo queue uploads when connectivity returns.

Offline: Service Worker registration, IndexedDB persistence via Zustand, photo queue with automatic retry. The iPad flow works without connectivity.

CX Review: Review page for CX after installer completes CM. DC context alert (shows when photos exist even without text notes — a subtle but important UX call). Save feedback displayed inline. API errors surfaced explicitly rather than failing silently.

PDF: Quote PDF parsing from InSite output. FormData upload for photos (not JSON — this was a fix, JSON encoding of binary data was causing silent failures). Customer signature captured at CM completion.

Installation: Installer flow for the final visit. All CM context available. Photos, status updates, completion.

**Branding:** Hillarys brand guidelines applied across all pages — matching the Ops OS visual identity. Login, header, colour system. Same team, same app family, same look.
